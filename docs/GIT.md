# Git 커밋 메시지 컨벤션

## 형식

```
<type>: <제목>

[본문 - 선택]
```

---

## Type 종류

| Type       | 설명                              | 예시                                      |
| ---------- | --------------------------------- | ----------------------------------------- |
| `feat`     | 새로운 기능 추가                  | `feat: 예약 등록 API 추가`                |
| `fix`      | 버그 수정                         | `fix: 강의실 중복 체크 오류 수정`         |
| `refactor` | 기능 변경 없는 코드 개선          | `refactor: ReservationService 메서드 분리` |
| `style`    | 포맷, 공백 등 코드 스타일 변경    | `style: 들여쓰기 정리`                    |
| `docs`     | 문서 추가 / 수정                  | `docs: API 명세 업데이트`                 |
| `chore`    | 빌드 설정, 패키지, 환경 변경      | `chore: springdoc 버전 업그레이드`        |
| `deploy`   | 배포 관련 설정 변경               | `deploy: GitHub Actions 워크플로우 추가`  |
| `test`     | 테스트 추가 / 수정                | `test: 예약 서비스 단위 테스트 추가`      |

---

## 규칙

- 제목은 **50자 이내**
- 제목 끝에 **마침표 금지**
- 제목은 **명령형**으로 작성 (`추가했다` X → `추가` O)
- 본문이 필요할 경우 제목과 **한 줄 띄우기**

---

## 예시

```
feat: 예약 등록 / 수정 / 삭제 API 추가

강의실, 숙박, 식수를 포함한 예약 전체 CRUD 구현
```

```
fix: 로그인 시 토큰 만료 오류 수정
```

```
chore: .gitignore에 application-security.properties 추가
```
